# コミット

以下を実行してほしい。

1. **状態確認**: `git status` で変更ファイルを確認する。
2. **ステージ**: コミット対象を決め、`git add` でステージする（全部でよければ `git add .`）。
3. **コミット**: `git commit` を実行する。
   - ユーザーがコミットメッセージを指定していればそれを使う。
   - 指定がなければ、変更内容（diff）から日本語で簡潔なメッセージを1行で提案し、それでコミットする。

注意:
- コミット前に `git status` と必要なら `git diff` の結果を確認してから進める。
- メッセージは「何をしたか」が分かるように書く（例: `docs: ブログにサンプル記事を追加`、`fix: mkdocs.yml のナビを更新`）。
